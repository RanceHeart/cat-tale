## 第一条死罪：旗子到不了

### 原因分析
看看 Level 1 地图：
```
row 2: "       %  ...  F "  ← 旗子在 row 2（天空层）
row 5: "   B...B      "     ← 砖块在 row 5
```
旗子放在 row 2（y=80），猫最高跳 ~88px，从 row 5 砖块（y=200）跳到 flag（y=80-28=52），需要跳 148px → **物理上不可达**。

### 修复
- 把 F 移到 row 4 或 row 5 同一行，且脚下有平台
- 在旗子下方加一级台阶（B 砖块），让猫能跳上去触碰到
- 旗子碰撞体加大（`flag.body.setSize(36, 96)`），不要那么苛刻
- 接近旗子时加一个闪烁/箭头提示，让玩家知道这是目标

## 第二条死罪：动作没有张力

### 原因
V2 的移动就是方方正正的左右+跳跃，没有任何视觉冲击。

### 修复清单

**1. 屏幕震动**
- 踩敌人时：轻度震屏（intensity: 4, duration: 100ms）
- 落地时（着地检测）：极轻度震屏（intensity: 2, duration: 60ms）
- 受伤时：中度震屏（intensity: 8, duration: 200ms）
- 过关时：庆祝震屏（intensity: 3, duration: 300ms）
- Phaser 方法：`this.cameras.main.shake(duration, intensity)`

**2. 粒子效果（落地灰尘）**
- 猫落地时从脚下喷出 6-8 个棕色小方块，向两侧扩散，0.4s 后消失
- 猫急停时也有小型灰尘
- 用 Phaser 的 `this.add.particles()` 或手动创建 sprite 组

**3. 跳跃预备（Anticipation，这是最重要的！）**
- 起跳前猫蹲一下：scaleY 从 1 → 0.85（0.06s），然后弹出去
- 这 0.06s 的蹲是手感的关键——让玩家感觉猫在蓄力
- 用 `this.tweens.add()` 实现，别用 setInterval

**4. 落地 squash**
- 着地瞬间：猫压扁（scaleY=0.75, scaleX=1.2），0.1s 弹回
- 已经有类似的代码但可能没生效，检查 tween 是否在正确时机调用

**5. 加速时镜头动态**
- 跑步时镜头微 zoom out（scale 1→1.02），停下恢复
- 或者：根据 cat velocity X 的绝对值，轻微改变 camera 的 lerp（更灵敏 = 更紧张）

**6. 吃鱼特效**
- 鱼被吃时：爆成一圈金色粒子（6个），向外飞散并淡出（0.3s）
- 加分文字 "+1" 从猫头顶弹出，上飘 + 淡出（1.0s）
- 鱼图标在 HUD 上有弹跳动画

**7. 踩敌人特效**
- 被踩的敌人先压扁（scaleY=0.4, scaleX=1.3, 0.08s）
- 然后缩成 0 消失（0.12s）
- 猫弹跳时加一个小弹跳粒子

## 第三条死罪：地图太单调

### 原因
V2 的地图就是平地 + 几个砖块 + 1个敌人，毫无设计感。

### 修复

**Level 1 "花园" — 教学关：**
```
"                                                                        ",
"                                                                        ",
"        %                %                %           F                ",
"                                                                        ",
"  @                                                                     ",
"   B====B   B====B        B==B    B====B     B====B    B======B         ",
"               %                    %                              %    ",
"=====    ===========    ========    ==========  ==============          ",
"                                    ^                           ^      ",
"=====  ========================================================   ======="
```
关键改动：
- 旗子在 row 2 的砖块正上方（站在砖块上跳到旗子）
- 第一段平地让玩家学走路
- 沟壑让玩家学跳跃
- 砖块上放鱼引导玩家往上跳
- 最后一段放两个敌人练习踩

**Level 2 "屋顶" — 进阶关：**
- 垂直段落：有一段要连续向上跳 3 层砖块
- 移动平台段：踩 M 平台过宽沟
- 多个高度层次，不是全在一条线上
- 狗敌人 + 乌鸦混搭
- 屋檐装饰（用浅色 = 砖块做屋顶形状）

**Level 3 "皇宫" — 挑战关：**
- S（尖刺）要出现在合理位置，不是摆设
- 有一段连续跳尖刺 + 窄平台
- Boss 在关底前出现，而不是一开始就放
- 室内色调（暖黄紫）

## 第四条死罪：美术太少

### 原因
背景就是渐变天空 + 几个 parallax 方块，关卡只有 = 和 B 两种砖，没有任何装饰性元素。

### 修复

**1. 装饰图层（纯视觉，无碰撞）**
新增 2 个地图字符：
- `g` = 草/花装饰（放在 = 上面）：草绿色小块 + 随机小花（黄/粉/白）
- `t` = 树装饰（放在地面上）：树干（棕色窄条）+ 树冠（绿色圆块），只渲染不碰撞
- `c` = 云装饰（天空层）：白色软云朵，缓缓飘动

在 BootScene 生成这些纹理：
```
this.textures.addCanvas("grass_deco", generateGrassDeco());
this.textures.addCanvas("tree", generateTreeTexture());
this.textures.addCanvas("cloud", generateCloudTexture());
```

**2. 背景层次**
- 远山：用三角形或圆顶山形，深蓝色半透明，固定在远处
- 近山/建筑轮廓：比远山大，颜色更深，随相机缓慢移动
- 基于关卡主题变化：
  - Level 1: 绿色山丘 + 树
  - Level 2: 城市屋顶轮廓线 + 烟囱
  - Level 3: 宫殿柱子 + 旗帜轮廓

**3. 地面多样化**
- Level 1: 绿草顶（地面 = 用绿色 #7EC850 做顶，棕色 #8B5E3C 做底）
- Level 2: 灰色瓦片顶（#8AA1B1 顶，深灰 #5D6975 底）
- Level 3: 大理石纹（#C8A87C 顶，#6F4E37 底）

**4. 鱼动画**
- 鱼在原地上下浮动（sin 波，幅度 6px，周期 0.8s）
- 让收集物有生命感

**5. 旗子动画**
- 旗帜部分微微飘动（左右摆动，幅度 4px，周期 1.2s）
